9. 딕셔너리의 키
- 다음과 같은 딕셔너리 `a`가 있다.
```python
>>> a = dict()
>>> a
{}
```

- 다음중 오류가 발생하는 경우를 고르고, 그 이유를 설명해보자
```python
a['name'] = 'python'
a[('a',)] = 'python'
a[[1]] = 'python'
a[250] = 'python'
```

<br>

- `a[[1]] = 'python'`코드에서 에러가 발생한다. 딕셔너리 타입의 Key로 설정할 수 있는 값은 가변 타입(mutable type)이 아니라 불변 타입(immutable type)이다. 따라서 가변타입인 리스트타입을 Key값으로 설정할 수 없다.

12. 파이썬 변수
- 파이썬은 다음처럼 동일한 값에 여러 개의 변수를 선언할 수 있다. 다음과 같이 a, b 변수를 선언한 후 a의 두번째 요소값을 변경하면 b값은 어떻게 될까? 그리고 이런 결과가 나타나는 이유를 설명해보자.

```python
a = b = [1, 2, 3]
a[1] = 4
print(b)
```

<br>

- `[1, 4, 3]`이 출력된다. 첫번째 행에서 리스트 객체 [1, 2, 3]이 생성되고 그 객체의 메모리 주소가 각각 b와 a에 동일하게 저장된다. 그래서 a가 가리키는 객체 index 1에 4를 대입하면 원본 리스트 객체가 [1, 4, 3]으로 변경되고 b는 a와 동일하게 여전히 이 객체를 가리키고 있다. 따라서 b를 프린트해도 변경된 리스트 객체가 나온다.
